STP 生成树

一、解决冗余环境的三大问题

1、广播风暴

2、数据帧多个副本

3、MAC地址的不稳定性

二、标准

1、原始STP：802.1D

2、CST

3、PVST和PVST+

4、RSTP

5、MSTP

三、选举过程

①根桥：

比较网桥ID（优先级和背板MAC地址），小的优

②根端口：

在非根桥交换机上选举，有且只有一个；接受BPDU，开销、网桥ID、端口ID（端口优先级+序列号）

③指定端口：

在每一条链路上，发送或转发BPDU，根桥上每一个端口都是指定端口；开销、网桥ID、端口ID

④阻塞端口：

逻辑和是哪个阻塞，只接受BPDU

四、端口角色

①disable：shutdown

②阻塞端口：只接受BPDU

③根端口：接受BPDU

④指定端口：发送或转发BPDU

五、端口状态

①down:shutdown

②blocking:阻塞状态

如果在20S内没有接收到对端的BPDU

③listening：（侦听状态）进行端口的状态转变

经历15S

④learning：学习状态，学习MAC地址

经历15S

⑤forwarding：可以正常转发数据帧

如果是本地链路出现故障，阻塞→转发30S
如果是非本地链路出现故障，阻塞→转发50S

六、BPDU的类型

BPDU：桥协议数据单元，用来协商STP的过程，发送周期2S，存活周期20S

①配置BPDU：存放网桥的相关信息（网桥ID，发送周期，老化时间等）

②TCN BPDU：拓扑变化BPDU，用来表示拓扑发生了变化

七、拓扑变化的过程

当拓扑发生变化，该交换机会往根桥方向发送一个TCN BPDU，根桥收到TCN BPDU，会回应TC BPDU，此时拓扑中的交换机收到TC的信息会将MAC地址的老化时间由300S降为15S，重新地址学习

Switch#show spanning-tree bridge
Switch#debug spanning-tree synchronization

八、STP的操作

（1）更改优先级

优先级默认是32768,（查看的优先级数字是由优先级+VLAN ID）

①更改优先级
SW2(config)#spanning-tree vlan 1 priority 2000
% Bridge Priority must be in increments of 4096.
% Allowed values are: 
  0     4096  8192  12288 16384 20480 24576 28672
  32768 36864 40960 45056 49152 53248 57344 61440
注意：必须是4096的倍数
②直接指定根桥
SW2(config)#spanning-tree vlan 1 root ?
  primary    Configure this switch as primary root for this spanning tree//指定为主根
  secondary  Configure switch as secondary root//指定为备根

（2）相关小特性

①portfast：对于接入交换机的接入端口，跳过30S的等待时间
SW3(config)#spanning-tree portfast edge default //全局配置
SW3(config)#int e0/2                            
SW3(config-if)#spanning-tree portfast 
*Jul  4 07:47:33.766: STP: VLAN0001 Et0/2 ->jump to forwarding from blocking

②uplinkfast：实现本地设备的链路出现故障，可以将原先的阻塞端口快速变为转发状态

SW3(config)#spanning-tree uplinkfast→会将接口的开销增加3000，以及优先级变为49152
*Jul  4 07:54:27.538: STP: VLAN0001 heard root  4097-aabb.cc00.1000 on Et0/1
*Jul  4 07:54:27.538:     supersedes 49153-aabb.cc00.3000
*Jul  4 07:54:27.538: STP: VLAN0001 new root is 4097, aabb.cc00.1000 on port Et0/1, cost 3100

③backbonefast：在所有交换机上使用，可以缩短20S的时间

SW1(config)#spanning-tree backbonefast

九、RSTP–802.1W

特点：能够快速收敛，向下兼容

十、端口角色

①RP

②DP

③AP：替代端口（备用端口），为RP备用

④BP：备份端口，为DP备份，但是只有在hub的情况下

十一、端口状态

①discarding：丢弃状态

②learning：学习状态

③forwarding：转发状态

十二、BPDU更改

保留原先的第一和第八比特，对中间的比特做了更改

十三、时间上

BPDU的发送周期还是2S，但是老化时间变为3倍的更新周期（6S）

十四、端口上

分为边缘和非边缘端口，边缘端口不会影响RSTP的稳定性，手工启动快速端口

十五、链路类型

①交换机与终端连接的端口（边缘端口）

→需要通过快速端口来快速过渡到转发状态

②交换机与交换机相连的端口（P2P）

→通过P/A机制来实现快速过渡到转发状态

③交换机与hub连接的链路（共享）

→无法通过P/A机制来实现快速过渡到转发状态

十六、P/A机制(提议/同意)

（1）提议/同意:

只能在P2P链路上使用，交换机A发送P比特置1的BPDU给交换机B，希望自己能快速过渡到转发状态，交换机B收到P比特置1的BPDU，会进入同步状态，交换机B会回应A比特置1的BPDU，交换机A收到A比特置1的BPDU，会过渡到转发状态，该过程与STP计时器无关

（2）同步状态：

将非边缘阻塞，避免环路

十七、拓扑变化

拓扑发生变化，交换机会直接发送TC位置1的BPDU，然后立即清空本地MAC地址表中的故障链路，其他交换机收到TC位置1的BPDU，也执行MAC地址重学习的过程（不需要发送TCN的BPDU）

十八、配置

Switch(config)#spanning-tree mode rapid-pvst

十九、MSTP

802.1S
特点：通过创建和维护实例来降低交换机的资源，实例中可以包含多个vlan，MSTP可以兼容多个版本STP

二十、基本概念

①CST：公共生成树 MST域与非MST域之间所运行的STP

②IST：内部生成树，运行MSTP

③MSTPI：MSTP内部生成树的实例，默认有实例0且所有的vlan都在其中

④CISTP：由CST和IST构成的整个交换网络的树

⑤总根：整个交换网络的根桥

⑥域根：是MSTP域内的根桥

⑦主交换机：距离总跟最近的交换机

二十一、配置

SW3(config)#spanning-tree mode mst
SW3(config)#spanning-tree mst configuration
SW3(config-mst)# name ccnp
SW3(config-mst)# revision 1
SW3(config-mst)# instance 1 vlan 10, 20, 30
SW3(config-mst)# instance 2 vlan 40, 50, 60
SW2(config)#spanning-tree mst 1 root primary →更改实例1的根桥

二十二、STP的优化

（1）BPDU guard

作用：如果在快速端口收到BPDU，则会将该端口关闭（err-disabled）

*Jul  6 03:06:48.486: %SPANTREE-2-BLOCK_BPDUGUARD: Received BPDU on port Ethernet0/2 with BPDU Guard enabled. Disabling port.
Sw2#
*Jul  6 03:06:48.486: %PM-4-ERR_DISABLE: bpduguard error detected on Et0/2, putting Et0/2 in err-disable state

（2）配置：

①Sw2(config)#spanning-tree portfast edge bpduguard →针对所有portfast接口
②Sw2(config-if)#spanning-tree bpduguard enable→强制将在接口下启用

err-disabled：错误状态 端口检测到错误的连接，会将端口自动关闭（shut-no shut）
bpdufilter（不建议）
作用：破坏快速端口的特性，进行正常的生成树端口选举
配置：

①Sw2(config-if)#spanning-tree bpdufilter enable 
②SW5(config)#spanning-tree portfast edge bpdufilter default rootguard

作用：保护根桥，如果在启用该特性的端口收到更优的BPDU，会将改端口进入blocking状态
配置：

SW2(config-if)#spanning-tree guard root

现象： 

*Jul  6 03:35:32.367: STP: VLAN0001 heard root  4097-aabb.cc00.5000 on Et0/2
*Jul  6 03:35:32.367:     supersedes 32769-aabb.cc00.1000
*Jul  6 03:35:32.367: %SPANTREE-2-ROOTGUARD_BLOCK: Root guard blocking port Ethernet0/2 on VLAN0001.
*Jul  6 03:35:32.367: STP: VLAN0001 sent Topology Change Notice on Et0/0
*Jul  6 03:35:32.368: STP[1]: Generating TC trap for port Ethernet0/2
*Jul  6 03:35:32.368: STP: VLAN0001 Et0/2 -> blocking loop guard

作用：检测单向链路所造成的环路问题
配置：

SW2(config)#spanning-tree loopguard default 
SW2(config-if)#spanning-tree guard loop

UDLD：光纤
Flexlink：不关心STP，只是对链路做主备
