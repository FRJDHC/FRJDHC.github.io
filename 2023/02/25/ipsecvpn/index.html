【IPSecVPN】

GRE：IP协议号47→轻量级的三层VPN（明文）
封装格式：IPHead（公网）+GRE+IPHead（私网头部）+DATA

（1）IPSec：不是一个具体的协议，而是多个安全协议构成的体系

（2）组件：安全协议、封装模式、IKE与安全联盟（SA）

（3）安全协议：

①AH：
a.特点：源认证，完整性检查和抗重放攻击，不支持加密，基于协议号51
②ESP：
a.特点：源认证，完整性检查和抗重放攻击且支持数据加密，基于IP协议号50
③ESP与AH的差别
a.AH不支持加密
b.AH验证整个数据包包含IP头部，ESP不需要验证IP报头，但是数据包在传递的过程中因为TTL值的变化，甚至在NAT环境下，源IP的变化，导致AH无法使用，所以ESP比AH更广泛被运用

（4）封装模式

①传输模式：IPSec插入在IP头部与数据之间
封装格式：IPH+IPSec+DATA
②隧道模式：IPSec封装在原始数据头部，并且在IPSec前面再生成新的IP头部
封装格式：IPH（新）+IPSec+IPH（原始）+DATA
区别：隧道模式会生成新的头部
③如何选择封装模式？
a.通信点：感兴趣流量（IPSecVPN要加密的流量）的产生设备
b.加密点：IPSecVPN架设的设备
c.通信点和加密点在同一个设备上→传输模式（节约报头）
d.通信点和加密点不在同一个设备上→隧道模式（借用新封装的头部，实现外网可达）→GRE

（5）IKE：网络密钥交换

①版本：IKEv1和IKEv2（FlexVPN）

②作用：提供IPSecVPN的密钥交换和建立安全联盟的服务 基于UDP端口号500

③组件：

a.Oakley：使用DH生成密钥
b.SKEME：验证密钥交换
c.ISAKMP：使用什么DH、什么加密算法、什么完整性检查等策略协商

④SA：基于IPSec双方某一些策略的约定，达成共识

a.三元组：SPI、目的地址、安全协议号
b.IKE SA：协商建立IPSec所使用的安全策略（协商采用何种安全策略）
c.IPSec SA：协商对用户的实际数据如何加密
d.SA是单向

⑤SPI：安全参数索引

a.使用一串数字标识SA→类似目录
b.由于SA是单向的，所以SPI也是单向
c.本地的出方向SPI等于对端的入方向SPI

⑥流程

a.IKE SA
1)模式的分类
主模式：6个数据包→三次交换（12,34,56）
第一次交互：协商策略→双方本地是否有相同的安全策略
第二次交互：密钥生成→生成双方加密算法使用
第三次交互：验证对等体→验证前面两次交互生成的安全策略和密钥→第5个数据包开始就被加密了
野蛮模式：3个数据包
b.IPSec SA
快速模式：3个包→12包协商保护用户数据的算法和密钥，3包为确认信息

【IPSecVPN的基本配置】

一阶段→IKE SA

R1(config)#crypto isakmp policy 10
R1(config-isakmp)#encryption 3des 
R1(config-isakmp)#authentication pre-share 
R1(config-isakmp)#group 5
R1(config-isakmp)#lifetime 43200

预共享密钥→cisco23（验证对等体）

R1(config)#crypto isakmp key cisco123 address 202.101.23.2
R1(config)#access-list 100 permit ip 192.168.10.0 0.0.0.255 192.168.20.0 0.0.0.255

二阶段→IPSec SA
感兴趣流量

R1(config)#crypto ipsec transform-set zwh esp-3des esp-sha-hmac 
R1(cfg-crypto-trans)#mode tunnel

关联一阶段和二阶段

R1(config)#crypto map zwh 10 ipsec-isakmp 
R1(config-crypto-map)#set peer 202.101.23.2
R1(config-crypto-map)#set transform-set zwh
R1(config)#int e0/1
R1(config-if)#crypto map zwh

验证命令：

R1#show crypto isakmp policy 
R1#show crypto map 
R1#show crypto isakmp sa 
R1#show crypto session 
R1#clear crypto sa

问题：

①如何实现异地站点的多内网地址互联？

问题：如果内网网段比较多，ACL的部署配置量太大也很繁琐→两个站点无法动态学习路由
解决方案：GRE+动态路由协议+IPSec（在这里只是为了加密，不提供连通性）
注意：ACL的书写不是写内网网段而是写GRE流量，还有VPN的封装模式建议使用传输模式→节省报头
access-list 100 permit gre host 202.101.12.1 host 202.101.23.2

②如果分支的IP地址不固定怎么办？

分支的代码无需变动，变动的是预共享密钥与静态map的邻居IP无法确定
解决方案：dynamic-map（动态map）
HUB的配置：

crypto isakmp policy 10
 authentication pre-share
crypto isakmp key cisco123 address 0.0.0.0 →由于spoke的公网IP未知，所以使用0.0.0.0       
crypto ipsec transform-set zwh esp-3des esp-sha-hmac 
 mode tunnel
crypto dynamic-map zwh 10→直接使用动态map调用转换集，不需要写ACL
 set transform-set zwh 
动态map无法直接调用在接收上，需要使用静态map关联动态map，然后接口再调用静态map
crypto map zwh 10 ipsec-isakmp dynamic zwh 
int e0/0
 crypto map zwh

弊端：总部无法主动发起连接

③有NAT环境又会出现什么情况？

场景一：VPN和NAT在同一台设备上
问题：主机能够访问外网，但是VPN中断
原因：由于NAT将内网的源IP改为了公网IP，导致数据包无法匹配VPN的ACL，因此不执行VPN操作
解决思路：SIP：192.168.10.1 DIP：192.168.20.1不要执行NAT的操作
配置：变更NAT的ACL（使用扩展ACL）

access-list 100 deny   ip 192.168.10.0 0.0.0.255 192.168.20.0 0.0.0.255→VPN流量不执
行NAT操作
access-list 100 permit ip 192.168.10.0 0.0.0.255 any→PC的上网流量执行NAT

场景二：VPN和NAT不在同一个设备上，VPN在NAT的后端
问题：分支的VPN邻居不在NAT上了→静态NAT
NAT穿越问题（NAT-T）：VPN设备会自动是被自己是否需要跨越NAT的设备，如果需要跨越，则会在网络层和ESP之间额外封装UDP，这是因为PAT需要端口号，但是ESP没有端口号无法进行NAT转换，因此UDP使用4500端口给VPN连接在NAT设备做端口映射

R1(config)#crypto ipsec nat-transparency udp-encapsulation →开启NAT-T功能

【IPSec profile】

作用：无需配置静态map，直接在ipsec profile上调用转换集，然后再隧道接口下调用ipsec-profile
配置：

crypto isakmp policy 10
 authentication pre-share
crypto isakmp key cisco123 address 202.101.23.2   
crypto ipsec transform-set zwh esp-3des esp-sha-hmac 
 mode transport
crypto ipsec profile zwh→创建IPSec profile
 set transform-set zwh 
interface Tunnel13→隧道接口上调用
 tunnel protection ipsec profile zwh

【ISAKMP profile】

作用：提高一阶段的效率，为一阶段提供配置的模板
配置：

crypto isakmp policy 10
 authentication pre-share
crypto keyring zwh  
  pre-shared-key address 202.101.23.2 key cisco123→预共享密钥
crypto isakmp profile zwh
   keyring zwh
   match identity address 202.101.23.2 255.255.255.255 →匹配对等体
crypto ipsec transform-set zwh esp-3des esp-sha-hmac 
 mode transport
crypto ipsec profile zwh
 set transform-set zwh
