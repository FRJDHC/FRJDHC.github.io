IPv6

网络工程 IPv6
笔记
发布日期:   2023-02-25
文章字数:   3.4k
阅读时长:   14 分

【IPv4的缺陷】

（1）地址范围2^32，地址空间不足
（2）NAT破坏了端到端的通信
（3）IPv4的报文结构比较复杂，设备处理报头比较繁琐，逐字段解析
（4）IPv4本身没有安全机制

【IPv6的特点】

（1）没有NAT
（2）地址范围：2^128
（3）一个接口可以有多个IPv6地址
（4）地址获取有三种方式：手工配置、无状态自动获取、有状态自动获取
（5）本身具备安全机制（AH、ESP的扩展报头）

【IPv6的报文格式】

IPv6的报头长度为40字节
（1）version
（2）traffic class：部署QoS
（3）flow label：类似FEC
（4）payload length：负载长度
（5）next header
（6）hop limit：TTL
（7）source address
（8）destination address
（9）option：扩展报头

【IPv6的扩展报头】

（1）hop by hop：必须每一台设备都需要处理
（2）routing：list中列出某台设备，那么数据包必须经过那台设备转发
（3）目标报头选项：目标节点的相应信息
（4）分片报头
（5）IP AH
（6）IP ESP
（7）上层报头：表示后续字段为上层字段

【IPv6地址】

（1）128比特→16字节
（2）冒号分16进制：共8组，一组16比特，4个16进制数表示，一个16进制数4比特
（3）IPv6的地址可以缩写，如果存在连续0（两组或者两组以上全为0）可以使用“::”省略，前导0可以用一个0代替
例子1：
完整：2021:417:0000:AABB:0000:0000:9876:1001
前导零省略：2021:417:0:AABB:0:0:9876:1001
::表示：2021:417:0000:AABB::9876:1001
例子2：
2021:417:0000:AABB:0000:0000:0000:1001
::表示：2021:417:0:AABB:0000::1001
例子3：
2021:0000:0000:AABB:0000:0000:9876:1001
省略：2021::AABB::9876:1001→错误（IPv6地址不能存在两个“::”）
正确的书写为：2021::AABB:0:0:9876:1001
例子4：
1::1→0001:0000:0000:0000:0000:0000:0000:0001
::→0000:0000:0000:0000:0000:0000:0000:0000

【IPv6地址类型】

（1）IPv6没有广播

（2）链路本地地址

→link-local address is FE80::A8BB:CCFF:FE00:100

①传输范围：

本链路有效→TTL=1

②特定接口：

由于物理接口可以存在多个IPv6地址，使得IPv6地址无法与接口一对一对应，使用链路本地地址标识特定接口

③作用：

路由协议的邻居建立、路由的下一跳等等

④如何产生？

前缀FE80::，然后在MAC地址中间插入FFFE，并且MAC地址的第七比特反转
例子：

aabb.cc00.0100
FE80::aabb:cc00:0100
FE80::AABB:CCFF:FE00:100
FE80::A1010BB:CCFF:FE00:100
FE80::A1000BB:CCFF:FE00:100
FE80::A8BB:CCFF:FE00:100
FE80::A8BB:CCFF:FE00:100

CD15.2021.0417→链路本地地址？
FE80::CD15:20FF:FE21:417
FE80::C 1101 15:20FF:FE21:417
FE80::C 1111 15:20FF:FE21:417
FE80::CF15:20FF:FE21:417
FE80::CF15:20FF:FE21:417

⑤链路本地地址：FE80::EUI标识符（CF15:20FF:FE21:417）

（3）全局单播地址

①前缀：2000::/3→（001）
②由IANA分配
③全局路由选择前缀、组织机构标识、接口ID

（4）独特的本地单播地址：类似IPv4的私有IP

①前缀：FC00::/7
②局域网内部使用，在互联网不可路由

（5）IPv4映射地址

①使用在6vPE环境，将IPv4地址映射为IPv6地址
②192.168.1.1→::FFFF:C0A8:101

（6）IPv6的组播地址

①前缀FF00::/8
②地址组成
FF00::/8→FF+flags+scope
FF固定
flags=0000，前面3个0固定，第四个比特位，如果为0表示永久，如果为1表示临时
scope：0000→表示使用的范围→E代表global

（7）组播MAC地址

①前缀：3333.XXXX.XXXX后32比特由被请求节点组播地址的后32位

（8）被请求节点组播地址

①前缀:FF02::1:FF/104+IPv6单播地址的后24位
②作用：DAD（重复地址检测）、获取邻居的MAC地址（NDP）

【IPv6地址的配置】

（1）手工配置

Router(config)#int e0/0
Router(config-if)#ipv6 enable →产生链路本地地址
Router(config-if)#int e0/1
Router(config-if)#ipv6 add fe80::2021:0503:1914 link-local →手工配置链路本地地址
Router(config-if)#ipv6 add 2021:503:1917::1/64→手工配置全局单播地址
Router(config-if)#ipv6 address 2021:503::/64 eui-64 →自己给定前缀，接口ID由EUI-64生成

（2）有状态自动获取→DHCPv6

①配置：

R1(config)#ipv6 dhcp pool DHCP
R1(config-dhcpv6)#address prefix 2020:ABCD:1234::/64
R1(config-dhcpv6)#dns-server 1::1
R1(config-dhcpv6)#domain-name www.yucedu.com
R1(config)#int e0/1
R1(config-if)#ipv6 dhcp server DHCP

客户端请求地址

Router(config-if)#ipv6 address dhcp

②基于UDP客户端546，服务端547

报文：solicit、advertise、request、reply

③DHCP中继：前提是路由可达

R2(config-if)#ipv6 dhcp relay destination 2020:12::1
R1#show ipv6 dhcp pool DHCP
R1#show ipv6 dhcp binding

④DHCP中继如何判断分配给客户端什么前缀的地址→GIA

⑤地址池分配的顺序？

1、MAC地址绑定
2、之前分配过的
3、从地址池中获取
4、从错误的地址状态中获取

【ICMPv6】

1、MAC地址解析

①IPv6没有ARP协议，数据的发送需要二层MAC地址的情况下，需要解析IPv6单播对应的MAC地址→NDP

②NS（135）和NA（136）

③工作原理:

HostA发送NS报文，源IP“hostA”目的IP“hostB的被请求节点组播地址”携带的内容：target：hostB的IPv6地址，option：“自己的MAC地址”
HostB发送NA报文，源IP“hostB”目的IP“hostA”携带的内容：hostB的MAC地址

2、DAD：重复地址检测

①主机在使用IPv6地址之前判断这个地址是否已经被使用→地址还未使用（试验地址）
②NS（135）和NA136
③工作原理：
HostA发送NS报文，源IP“：：”目的IP“试验地址所对应的被请求节点组播地址”携带内容：target：试验地址
a.无人回应时→HostA发送NA报文，源IP“试验地址”目的IP“FF02::1”携带内容：试验地址与本地MAC地址的映射
b.有人回应：如果hostB的地址也是试验地址，则会放弃使用并且不会发送NA报文；如果该地址已经正在使用，则会回应NA报文，hostA收到NA报文会放弃使用该试验地址

3、路由器发现（无状态自动获取）

①作用：

实现主机上线立刻获取IPv6地址（网络前缀、router lifetime、MTU）

②RS（133）和RA（134）

③工作原理：

hostA发送RS报文，源IP“链路本地地址”目的IP“FF02::2”携带的内容为hostA的MAC地址
RouterA发送RA报文，源IP“链路本地地址”目的IP“FF02::1”携带的内容为网络前缀、路由器等级、MTU、router lifetime
hostA拿到IPv6网络前缀+EUI-64生成IPv6地址并且进行DAD

④路由器等级：低级、中级（默认等级）、高级

R2(config-if)#ipv6 nd router-preference high 
ND  ::/0 [2/0]
     via FE80::A8BB:CCFF:FE00:210, Ethernet0/0
NDp 2020:ABCD:1234::/64 [2/0]
     via Ethernet0/0, directly connected

4、重定向

①作用：

在链路存在多台设备访问出现次优路径

②reject：type=137

③工作原理：

hostA发送数据包给网关，routerA收到数据包后，判断是否可以发送重定向报文：
a.目的地址不是一个组播地址
b.接收报文和发送报文在同一接口
c.源IP和最优的下一跳在同一网络前缀
d.源IP和最优下一跳在邻居表中存在
满足条件时，RouterA发送ICMP，type=137的reject报文，hostA收到重定向报文，在本地路由表中添加一条目标网段的主机路由，下一跳指向最优的下一跳

5、path MTU

①作用：探查源与目的之间的最小MTU

②ICMP：type=2→packet too big

③源以自己接口的PMTU值发送探查报文，如果链路存在接口小于该PMTU值，则回送ICMP差错报告（packet too big）源收到差错报文时，就已报文携带的PMTU作为探查报文的PMTU值，如此反复，检测链路的最小PMTU

①使用链路本地地址建立邻居
②进程可以关闭

#### ②基于IPV4邻居传输IPv6路由→使用路由图修改下一跳（全局单播路由）

#### ③基于IPv6传递IPv6路由→
注意下一跳R5(config-router)#ipv6 router eigrp 90→R5(config-rtr)#redistribute connected
YAML router bgp 200 bgp router-id 6.6.6.6 bgp log-neighbor-changes neighbor 5656::1 remote-as 100 ! address-family ipv4 no neighbor 5656::1 activate exit-address-family ! address-family ipv6 neighbor 5656::1 activate exit-address-family R6#show ip bgp ipv6 unicast summary R6#show ip bgp ipv6 unicast
### 5、OSPFv3
#### （1）需要32位的RID
#### （2）建立邻居使用链路本地地址→全局单播地址被作为一个叶子节点挂在路由器上
#### （3）支持多拓扑
#### （4）LSA的变化→将拓扑与路由分离
###### ①3、4、5、7LSA内容没有发生变化只是名称变了
###### ②1、2类LSA发生变化→只携带拓扑信息不携带网络前缀
###### ③新增8、9类LSA用于携带网络前缀
#### 1类LSA：
Router Link States→描述的是自己伪节点的拓扑信息
#### 拓扑类型：
transit、p2p、virtual-link
#### 2类LSA：
R3#show ipv6 ospf database network→描述的是链路上的拓扑信息，由DR产生
#### 8类LSA：
Link (Type-8) Link States 描述的是本地链路的所有前缀信息，每一台设备都会产生
#### 9类LSA：
Intra Area Prefix Link States→描述的是区域内各链路的所有前缀信息，如果是广播多路访问环境由DR产生，依靠1/2类LSA，如果是P2P,每一台路由器都会产生
##由以上LSA构建区域内的SPF树

3类LSA：

Inter Area Prefix Link States

4类LSA：

Intra Area Prefix Link States

5类LSA：

Type-5 AS External Link States

（5）配置：

router ospfv3 110
 router-id 6.6.6.6
 !
 address-family ipv6 unicast
  redistribute connected
 exit-address-family
interface Serial1/0
 ipv6 address 2021:56::2/64
 ipv6 ospf 110 area 2

（6）影响OSPF邻居建立的因数

①网络类型：

MA、P2P（10S 40S）、NBMA、P2MP（30S、120S）NBMA只能跟NBMA建立（思科上MA、NBMA就能建立→改时间）

②RID：

同区域直连不能一样，同区域非直连能够正常建立邻居，但是1LSA出现震荡，不同区域RID能够正常建立与计算路由，5类LSA震荡

③HELLO时间：

10S 40S

④接口ID：

区域号，邻居的区域号必须一致

⑤特殊区域：

stub不能与普通区域、NSSA、骨干区域不能是特殊区域

⑥认证：

无认证不能与有认证、明文认证不能与密文认证、认证密码必须一致

⑦MTU：

1500→MTU不一致会卡在exstart→（主1500 从1400， 主1400 从1500）

⑧掩码：

广播多路访问→DR→保持一致

⑨被动接口：

只收不发

【IPv6的过渡技术】

1、手工配置隧道

GRE隧道:IPv4 GRE IPv6

interface Tunnel24
 ipv6 address 2021:2424::2/64
 ipv6 ospf 110 area 0
 tunnel source Loopback1
 tunnel destination 2.2.2.2
IPv6 in IPv4
R2(config-if)#tunnel mode ipv6ip →直接将IPv6封装在IPv4中

2、自动隧道

6to4隧道：使用固定的前缀：2002::/16
interface Tunnel24
 ipv6 address 2021:2424::/64
 ipv6 address 2021:2424::1/64
 tunnel source Loopback1
 tunnel mode ipv6ip 6to4
ipv6 route 2002::/16 Tunnel24

3、ISATAP

工作原理：

隧道建立后，主机通过链路本地地址向FE80::5efe:4.4.4.4发送RS消息请求IPv6前缀，isatap服务器通过RA回应隧道前缀，主机通过前缀+::5efe+本地IPv4地址生成IPv6地址，将isatap服务器作为网关

interface Tunnel1
 ipv6 address 2021:505::1/64
 tunnel source Loopback1
 tunnel mode ipv6ip isatap

Windows 7/10配置

netsh interface isatap set state default
netsh interface isatap set state enable
netsh interface isatap set router 4.4.4.4

注意：ISTAP隧道无法跨越NAT

实现通过标签转发IPv6路由

::FFFF:2.2.2.2 下一跳不可达→通过标签解决

router bgp 100
 bgp log-neighbor-changes
